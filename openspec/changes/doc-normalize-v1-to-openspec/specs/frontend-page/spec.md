# Spec: 前端页面交互规则

## 概述

本 spec 定义新点 SaaS 造价系统前端页面与交互的稳定规则，用于统一项目管理页、阶段工作台、清单页、定额页、汇总页和报表页的页面目标、核心区域和关键交互边界。

## ADDED Requirements

### Requirement: 项目列表页只能展示用户可访问项目

项目列表页必须只展示当前用户具备访问权限的项目，并优先突出其待处理或待审核事项。

#### Scenario: 查看项目列表

- **GIVEN** 用户进入项目列表页
- **WHEN** 系统加载项目数据
- **THEN** 页面只应展示该用户可访问的项目

### Requirement: 项目详情页必须作为统一入口

项目详情页必须作为项目的统一入口，承载阶段导航、核心业务入口、成员权限入口和版本摘要。

#### Scenario: 打开项目详情

- **GIVEN** 用户进入项目详情页
- **WHEN** 页面渲染完成
- **THEN** 用户应能在同一入口中切换阶段工作台和进入核心业务页面

### Requirement: 阶段工作台必须展示阶段摘要、待办和风险

阶段工作台必须围绕当前阶段展示阶段摘要、待办事项、核心入口、数据引用摘要和风险提示。

#### Scenario: 查看当前阶段工作台

- **GIVEN** 用户处于某项目阶段工作台
- **WHEN** 页面展示阶段信息
- **THEN** 应同时呈现阶段状态、负责人、待办、引用数据和风险提示

### Requirement: 清单页必须显式展示版本与来源

清单编制页必须显式展示当前阶段、当前版本、来源版本、锁定状态、创建人和变更原因，避免用户误将不同阶段版本视为同一可覆盖对象。

#### Scenario: 查看清单主页面

- **GIVEN** 用户打开清单页
- **WHEN** 页面展示当前版本信息
- **THEN** 页面应显式展示版本与来源信息

### Requirement: 清单页交互必须围绕工具栏、树、表格、侧栏和状态栏组织

清单主页面必须至少由顶部工具栏、左侧结构树、中央表格区、右侧详情侧栏和底部状态栏组成。

#### Scenario: 进入清单编辑

- **GIVEN** 用户进入清单编辑页面
- **WHEN** 页面完成布局渲染
- **THEN** 页面应提供上述核心区域

### Requirement: 不同阶段必须复用同一清单页面模型但裁剪可编辑范围

招标清单、投标报价、合同清单、施工过程变更清单和竣工结算应复用同一清单页面模型，但字段可编辑范围和限制规则必须按阶段裁剪。

#### Scenario: 合同清单页面

- **GIVEN** 用户打开合同清单页面
- **WHEN** 版本已经锁定
- **THEN** 修改类按钮应默认禁用

### Requirement: 定额页必须在清单上下文中工作

定额套用页必须在清单上下文中工作，并围绕清单摘要、工具栏、定额表格、AI 推荐和详情侧栏组织。

#### Scenario: 打开定额页面

- **GIVEN** 用户从清单上下文进入定额页
- **WHEN** 页面渲染
- **THEN** 用户应能看到当前清单摘要和关联定额上下文

### Requirement: 汇总页必须支持多维筛选和风险提示

汇总页面必须支持按项目、阶段、专业、单体、版本等维度筛选，并展示核心指标、汇总表、图表和风险提示。

#### Scenario: 查看汇总页

- **GIVEN** 用户进入汇总页
- **WHEN** 页面加载完成
- **THEN** 页面应同时展示筛选区、核心指标和汇总结果

### Requirement: 报表页必须区分任务列表和模板管理

报表中心必须区分报表列表和报表模板管理，分别承载生成记录、状态查看和模板维护。

#### Scenario: 打开报表中心

- **GIVEN** 用户进入报表中心
- **WHEN** 页面呈现
- **THEN** 用户应能区分报表任务列表和模板管理入口

### Requirement: 前端按钮显隐必须受权限和状态共同约束

页面中的提交、审核、锁定、解锁、导入、导出和 AI 相关按钮，必须同时受权限和状态约束。

#### Scenario: 查看提交按钮

- **GIVEN** 用户处于某阶段页面
- **WHEN** 页面决定是否展示“提交审核”
- **THEN** 该按钮必须同时满足用户具备相应权限且当前对象处于可提交状态

## Implementation Notes

以下内容是当前仓库实现现状的观察，不属于规范要求本身：

- 当前仓库 `apps/frontend` 已有项目页、清单页、汇总页和若干共享组件骨架。
- 现有前端实现仍是可运行工作台与测试基线，尚未完全覆盖设计文档中的全部页面能力。
- 当前文档中的页面规则远比现实现状更完整，因此实现不应反向收缩规范边界。

这些现状说明前端页面主结构已经起步，但完整中后台交互体系仍属于后续工程工作。
