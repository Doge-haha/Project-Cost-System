# Spec: 项目工作台专题

## 概述

本 spec 定义新点 SaaS 造价系统中的项目工作台专题规则，用于统一项目统一入口、阶段切换、摘要聚合、快捷入口和侧边栏信息结构。

## ADDED Requirements

### Requirement: 项目工作台必须作为进入项目后的统一入口

用户进入项目后，应先进入统一工作台，再从中切换阶段、查看摘要和进入具体业务模块。

#### Scenario: 进入项目

- **GIVEN** 用户打开某项目
- **WHEN** 系统加载项目主页
- **THEN** 应展示项目工作台作为统一入口

### Requirement: 工作台必须按启用阶段组织导航

工作台必须基于项目已启用阶段组织左侧导航和当前阶段视图，不能脱离阶段模板独立展开。

#### Scenario: 切换阶段

- **GIVEN** 项目已启用多个阶段
- **WHEN** 用户在工作台切换阶段
- **THEN** 系统应按启用阶段显示导航和当前阶段视图

### Requirement: 工作台必须聚合项目、阶段、专业和权限摘要

工作台必须至少聚合项目基础信息、当前阶段摘要、启用阶段列表、专业配置摘要和当前成员权限摘要。

#### Scenario: 查看工作台摘要

- **GIVEN** 用户进入工作台
- **WHEN** 系统返回聚合数据
- **THEN** 应能看到项目、阶段、专业和权限摘要

#### Scenario: 返回当前用户权限摘要

- **GIVEN** 当前用户进入项目工作台
- **WHEN** 系统返回工作台聚合结果
- **THEN** 结果中应包含当前用户角色、可管理能力、可编辑能力以及可见阶段与专业范围摘要

### Requirement: 工作台必须预留待办、风险和导入状态摘要

工作台应预留待办、风险、导入状态或其他全局提示字段，作为后续消息和流程联动的统一承载位。

#### Scenario: 显示工作台提示

- **GIVEN** 项目存在待办、风险或导入相关提醒
- **WHEN** 用户查看工作台
- **THEN** 系统应能在统一摘要区展示这些信息

#### Scenario: 导入状态基于正式导入任务模型返回

- **GIVEN** 项目已经发起过导入或知识提取任务
- **WHEN** 工作台返回 `importStatus`
- **THEN** 系统应基于正式导入任务模型汇总总数、排队中、处理中、已完成、失败以及最近导入任务摘要

### Requirement: 工作台右侧信息栏必须承载最近操作和状态摘要

工作台右侧侧栏必须承载审核状态、最近操作、消息提醒和版本摘要等高频辅助信息。

#### Scenario: 查看侧边栏

- **GIVEN** 用户位于项目工作台
- **WHEN** 系统渲染右侧信息区
- **THEN** 应展示审核状态、最近操作、消息提醒或版本摘要

#### Scenario: 通过时间线查看最近动态

- **GIVEN** 项目内已经发生审核、过程单据或任务处理动作
- **WHEN** 用户打开项目工作台
- **THEN** 右侧信息区应能按时间倒序展示最近动态，帮助用户快速判断项目刚刚发生了什么

#### Scenario: 从最近动态返回对应处理上下文

- **GIVEN** 项目工作台已经展示最近动态时间线
- **WHEN** 用户点击某条审核、过程单据或任务动态
- **THEN** 系统应跳回对应处理页，并保留当前项目上下文

#### Scenario: 最近动态应支持复制当前处理链接

- **GIVEN** 项目工作台已经展示可跳转的审核、过程单据或任务动态
- **WHEN** 用户复制某条动态对应的处理链接
- **THEN** 系统应返回可直接分享的处理页链接，帮助协作同事进入同一处理上下文

#### Scenario: 复制处理链接后应支持立即打开该入口

- **GIVEN** 用户刚在项目工作台最近动态或待办页列表中复制了一条处理链接
- **WHEN** 页面展示复制成功反馈
- **THEN** 系统应提供一个可立即打开该处理入口的操作，方便用户继续核对刚复制的上下文

#### Scenario: 入口页应展示最近复制的处理入口

- **GIVEN** 用户刚在项目工作台、待办页或任务状态页复制过一条处理链接
- **WHEN** 用户继续在同一项目的这些入口页之间切换
- **THEN** 系统应展示最近复制入口，并允许用户重新打开或清除该记录，减少重复筛选和二次交接成本

#### Scenario: 批量处理后的最近协作摘要应支持展开对象清单

- **GIVEN** 用户刚完成一轮批量审核或批量过程单据处理，并返回项目工作台或待办页
- **WHEN** 用户收起刷新提示后查看最近协作摘要
- **THEN** 系统应默认保留一句批量处理总述，并允许用户在摘要区内折叠或展开本轮对象清单，直接回看每个对象的处理入口，同时明确标识对象所属类型

#### Scenario: 批量对象清单应支持直接复制单条入口

- **GIVEN** 用户已经在最近协作摘要中展开本轮对象清单
- **WHEN** 用户选择其中某一个审核或过程单据对象并复制该对象入口
- **THEN** 系统应直接返回该对象的可分享处理链接，并在摘要区内提供刚复制入口的反馈，同时记住本轮最近一次被复制的具体对象，减少再次进入对象详情页后再复制的往返操作

#### Scenario: 失败子集最近协作摘要也应支持对象清单

- **GIVEN** 用户已经在任务状态页把失败原因继续收束到更小子集，并把这组问题作为协作处理单元复制或回流到入口页
- **WHEN** 用户在项目工作台或待办页查看最近协作摘要
- **THEN** 系统也应支持把该失败子集中的具体失败记录作为对象清单展开展示，并明确标识这是失败记录对象，便于用户继续回看、复制或再次定位单条失败记录

#### Scenario: 再次进入入口页时应自动定位最近复制对象

- **GIVEN** 用户已经在批量对象清单中复制过某一个具体对象入口
- **WHEN** 用户随后切换到其他入口页，再回到当前项目的工作台或待办页
- **THEN** 系统应自动展开对象清单并高亮最近复制对象，帮助用户快速确认刚刚交接出去的是哪一条

#### Scenario: 可清除单条最近复制对象但保留整轮摘要

- **GIVEN** 用户已经在批量对象清单中复制过某一个具体对象入口，摘要区因此记住了最近复制对象
- **WHEN** 用户结束这条单独对象的交接，并在摘要区清除最近复制对象
- **THEN** 系统应只移除单条对象的高亮和最近复制提示，继续保留整轮批量摘要和对象清单；如果对象清单只是因最近复制对象而自动展开，则应恢复默认折叠，如果用户已手动展开对象清单，则应保持当前展开状态

#### Scenario: 无最近复制高亮时应恢复用户上次手动展开偏好

- **GIVEN** 用户曾在最近协作摘要中手动展开或收起过本轮对象清单，且当前摘要没有最近复制对象高亮
- **WHEN** 用户切换到其他入口页后再返回同一项目的最近协作摘要
- **THEN** 系统应恢复用户最近一次手动展开偏好，避免每次都重新展开或收起同一轮对象清单

#### Scenario: 切换到新一轮批量摘要时不应沿用上一轮展开偏好

- **GIVEN** 用户上一轮批量摘要曾记住手动展开或收起偏好
- **WHEN** 系统写入同一项目的新一轮批量摘要
- **THEN** 新一轮摘要应恢复默认折叠状态，不应继承上一轮对象清单的展开偏好，避免旧的浏览习惯误带到新的协作上下文

#### Scenario: 批量摘要应标识本轮来源入口

- **GIVEN** 用户正在查看最近协作摘要中的一轮批量处理记录
- **WHEN** 用户查看摘要总述或展开对象清单前的概览信息
- **THEN** 系统应明确标识本轮记录来自待办页、项目工作台或其他协作入口，并以 `回到来源页` 的明确动作文案支持用户直接返回对应来源入口的筛选态；对于待办页、项目工作台这类包含分区或摘要上下文的入口，动作文案还应补充当前分区语义，帮助用户预判会回到待办、风险、导入，或对应的工作台摘要；如果当前已经位于该来源入口，则应改为提示用户已在当前上下文中，避免无效跳转

#### Scenario: 单条最近协作摘要也应展示来源入口动作

- **GIVEN** 用户正在查看一条由审核页、过程单据页或任务状态页复制出来的单条最近协作摘要
- **WHEN** 用户查看摘要区中的最近协作记录
- **THEN** 系统也应展示对应的来源入口动作，并在可识别筛选语义时补充如审核处理页的待处理、已驳回，过程单据页的待审核、草稿、已驳回，以及任务状态页的失败、处理中、已完成等上下文；这些语义应在跨页查看摘要时以明确动作文案保留，帮助用户快速返回原始处理入口

#### Scenario: 来源动作与最近协作入口不应重复呈现

- **GIVEN** 最近协作摘要已经能识别出一条明确的来源入口动作
- **WHEN** 用户当前并不在该来源页中查看摘要
- **THEN** 系统应将来源动作作为唯一主跳转入口，避免同时再展示一条作用重复的“打开最近协作入口”；只有当前已经位于来源页时，才保留“打开最近协作入口”用于重新打开这条精确上下文

#### Scenario: 来源动作与最近复制对象动作应区分主次层级

- **GIVEN** 最近协作摘要同时存在来源动作，以及“打开最近复制对象”或“打开刚复制入口”这类继续交接动作
- **WHEN** 用户快速扫读摘要区中的可点击动作
- **THEN** 系统应将“回到来源页”呈现为主动作，把最近复制对象和刚复制入口呈现为次动作，帮助用户优先判断是回到原始上下文还是继续打开单条交接对象

#### Scenario: 返回对应处理页时自动聚焦目标记录

- **GIVEN** 用户从最近动态进入审核、过程单据或任务处理页
- **WHEN** 页面完成加载
- **THEN** 系统应自动选中对应记录，并在可处理时直接展开动作区

#### Scenario: 处理完成后回到工作台显示刷新提示

- **GIVEN** 用户刚在审核、过程单据或任务页完成一次处理动作
- **WHEN** 用户返回项目工作台或待办页
- **THEN** 系统应显示轻量刷新提示，并让用户明确知道摘要和最近动态已经刷新

### Requirement: 工作台摘要应提供可继续处理的入口

工作台中的待办、风险、导入状态等摘要不应停留在只读展示，而应提供进入后续处理页或明细页的快捷入口。

#### Scenario: 从摘要进入待办页

- **GIVEN** 用户正在查看项目工作台摘要
- **WHEN** 用户点击待办、风险或导入状态摘要入口
- **THEN** 系统应进入对应的处理页或明细页，并保留当前项目上下文

#### Scenario: 待办页列表项应支持复制处理入口

- **GIVEN** 用户正在待办页查看审核待办或过程单据列表
- **WHEN** 用户复制某条列表项对应的处理链接
- **THEN** 系统应返回可直接分享的处理页链接，帮助协作同事进入同一处理上下文

#### Scenario: 从待办页进入审核处理

- **GIVEN** 项目工作台待办页已展示审核待办
- **WHEN** 用户进入审核处理页并执行通过、驳回或撤回
- **THEN** 系统应完成审核动作，并让用户在当前项目上下文内继续处理剩余待办

#### Scenario: 审核处理页应保留可分享的当前处理视角

- **GIVEN** 用户已进入审核处理页并选中某条审核记录的通过、驳回或撤回动作
- **WHEN** 页面进入具体处理态、用户取消处理，或当前 URL 指向不存在/不可执行的审核动作
- **THEN** 系统应分别写入、清理或纠正 `reviewId` 与 `action` 参数，让当前处理视角能够被稳定回放和分享

#### Scenario: 从待办页进入过程单据处理

- **GIVEN** 项目工作台待办页已展示过程单据风险或待处理状态
- **WHEN** 用户进入过程单据处理页并执行提交、通过或驳回
- **THEN** 系统应完成过程单据状态流转，并让用户在当前项目上下文内继续处理剩余事项

#### Scenario: 过程单据处理页应保留可分享的当前处理视角

- **GIVEN** 用户已进入过程单据处理页并选中某条单据的提交、通过或驳回动作
- **WHEN** 页面进入具体处理态、用户取消处理，或当前 URL 指向不存在/不可执行的单据动作
- **THEN** 系统应分别写入、清理或纠正 `documentId` 与 `action` 参数，让当前处理视角能够被稳定回放和分享

#### Scenario: 从工作台进入任务状态并重试失败任务

- **GIVEN** 项目工作台已展示导入状态或异步任务摘要
- **WHEN** 用户进入任务状态页查看失败原因并重试失败任务
- **THEN** 系统应在当前项目上下文内展示任务详情，并允许有权限的成员把失败任务重新入队

#### Scenario: 发起导入任务后工作台与任务状态共享同一导入记录

- **GIVEN** 用户通过项目内导入入口或知识提取入口发起一次导入
- **WHEN** 系统创建导入任务并排入异步处理
- **THEN** 工作台摘要与任务状态页应引用同一条导入任务记录，而不是分别维护独立状态来源

#### Scenario: 导入任务应返回来源文件、批次和失败明细

- **GIVEN** 某次导入来自文件上传或系统批量导入
- **WHEN** 用户在任务状态页查看导入任务
- **THEN** 系统应返回来源文件名、批次号、失败明细以及当前重试次数与上限，方便用户判断是否继续重试

#### Scenario: 用户上传导入文件后应自动生成导入批次

- **GIVEN** 用户位于项目任务状态页并选择一个支持的导入文件
- **WHEN** 用户发起上传
- **THEN** 系统应解析文件内容、创建新的导入任务批次并排入异步处理，同时在任务状态页展示该批次

#### Scenario: 任务状态页应能展开导入批次详情

- **GIVEN** 项目中已经存在多个导入批次
- **WHEN** 用户在任务状态页选择某个导入任务
- **THEN** 系统应展示该批次的来源、格式、文件名、批次号、失败摘要、关联任务和批次元数据

#### Scenario: 批次详情应能下钻查看解析摘要与重试历史

- **GIVEN** 某个导入批次来自真实文件上传或已经历过重试
- **WHEN** 用户在任务状态页查看该批次详情
- **THEN** 系统应展示解析概览、条目预览以及重试历史，帮助用户判断这批数据的结构和后续处理轨迹

#### Scenario: 批次详情应能按失败原因筛选问题条目

- **GIVEN** 某个导入批次已经返回失败原因归类和失败条目明细
- **WHEN** 用户在任务状态页切换某个失败原因
- **THEN** 系统应只展示对应失败条目，并明确当前协作视角

#### Scenario: 协作视角应能跨页面保留

- **GIVEN** 用户已经围绕某个失败原因进入任务状态页、待办页或项目工作台
- **WHEN** 用户在这些页面之间来回跳转
- **THEN** 系统应保留同一失败原因视角，而不需要用户重新筛选

#### Scenario: 失败子集协作单元应跨页面保留

- **GIVEN** 用户已经在任务状态页中围绕某个失败原因继续收束到更小的失败子集，例如资源类型或动作
- **WHEN** 用户从任务状态页回到项目工作台或待办页，并再次进入任务状态页
- **THEN** 系统应继续保留该失败子集协作单元，而不是只退化为更粗粒度的失败原因视角；入口页也应显式展示当前协作处理单元，帮助用户确认当前正在跟进的范围

#### Scenario: 导入任务未加载完成前不应误删失败子集参数

- **GIVEN** 用户通过带有 `failureReason`、`failureResourceType`、`failureAction` 的任务状态页链接进入页面
- **WHEN** 页面仍在等待导入任务详情和失败条目摘要加载完成
- **THEN** 系统不应在初始化阶段就把这些子筛选参数判定为无效并从 URL 中移除；只有在导入任务加载完成后仍确认参数不属于当前批次的失败条目范围时，才允许自动清理，避免用户原本有效的失败子集协作单元在首屏加载时被意外降级

#### Scenario: 任务状态页应兼容正式失败快照结构

- **GIVEN** 导入任务除了 `failedItems` 之外，还通过独立的 `failureSnapshots` 元数据返回了可重试的失败记录快照
- **WHEN** 用户在任务状态页按失败原因、资源类型或动作收束到某个失败子集
- **THEN** 系统应把这些正式快照合并回当前失败记录，用于判断该子集是否具备可重建输入，并在导出当前子集时保留对应快照，而不要求后端继续把同一份快照重复内联到每条 `failedItems` 中

#### Scenario: 导出当前失败子集时应显式附带失败快照摘要

- **GIVEN** 当前失败子集中的部分或全部条目已经具备可重建输入快照
- **WHEN** 用户在任务状态页导出当前子集 JSON
- **THEN** 导出内容除了 `failedItems` 之外，还应显式带上与当前子集对应的 `failureSnapshots` 摘要，便于后续处理方直接消费正式快照结构，而不需要再从失败条目中二次提取

#### Scenario: 复制失败子集协作摘要时应说明可重建快照数量

- **GIVEN** 用户准备把当前失败原因或失败子集交接给协作同事或上游数据方
- **WHEN** 用户在任务状态页复制协作摘要或失败子集处理单
- **THEN** 复制内容应显式说明当前范围内具备可重建输入快照的条目数，帮助接收方快速判断这批问题更适合直接子集重试，还是继续走回源修数流程

#### Scenario: 页面摘要与处理单应展示可重建快照数量

- **GIVEN** 用户正在任务状态页查看某个失败原因或已收束的失败子集
- **WHEN** 页面展示当前筛选运营摘要或当前失败子集处理单
- **THEN** 页面应显式展示当前范围内具备可重建输入快照的条目数，确保页面阅读、复制摘要和导出 JSON 三者对同一份快照能力保持一致

#### Scenario: 单条回看应提示当前条目是否具备可重建快照

- **GIVEN** 用户已经在任务状态页进入某条失败记录的单条回看
- **WHEN** 当前条目本身已经具备可重建输入快照
- **THEN** 单条回看区域应明确提示这条记录可直接纳入当前子集重试，避免用户还要回到列表或摘要区重新确认该状态

#### Scenario: 协作视角下进入任务状态应默认聚焦失败列表

- **GIVEN** 用户已从项目工作台或待办页带着某个失败原因视角进入任务状态页
- **WHEN** 系统打开任务状态页
- **THEN** 应默认落在失败任务视图，减少重复筛选动作

#### Scenario: 协作视角应支持复制与手动清除

- **GIVEN** 用户当前处于某个失败原因的协作视角
- **WHEN** 用户复制当前链接或手动清除该视角
- **THEN** 系统应分别支持分享当前问题视角和回到普通视图

#### Scenario: 任务状态页应支持复制当前处理入口

- **GIVEN** 用户已经把失败条目收束到当前子集，或进入了单条失败记录回看
- **WHEN** 用户复制当前处理链接
- **THEN** 系统应返回包含当前筛选或单条定位参数的处理页链接，帮助协作同事进入同一处理上下文

#### Scenario: 任务状态页复制链接后应支持立即打开

- **GIVEN** 用户刚在任务状态页复制了当前筛选链接或当前处理链接
- **WHEN** 页面展示复制成功反馈
- **THEN** 系统应提供一个可立即打开该处理入口的操作，方便用户继续核对刚复制的上下文

## Implementation Notes

以下内容是当前仓库实现现状的观察，不属于规范要求本身：

- 当前 TypeScript API 已补充 `GET /v1/projects/:projectId/workspace` 聚合接口，用于收敛项目、当前阶段、可用阶段、已启用专业、可见版本以及当前用户权限摘要。
- 当前前端项目详情页已切换为优先消费该聚合入口，作为项目级 workspace 统一入口。
- 当前 TypeScript API 已在 workspace 结果中补充 `todoSummary`、`riskSummary` 与 `importStatus`。
- `todoSummary` 当前基于审核记录与过程单据状态聚合，`riskSummary` 当前基于驳回记录与失败异步任务聚合。
- 当前 TypeScript API 已补充正式 `import_task` 模型，工作台中的 `importStatus` 已改为基于导入任务聚合，而不再复用异步任务占位。
- 当前 TypeScript API 已补充 `GET /v1/projects/:projectId/import-tasks` 与 `POST /v1/projects/:projectId/import-tasks`，并让知识提取入队链路先创建导入任务，再创建异步任务。
- 当前 TypeScript API 已把来源文件名、批次号、失败明细、重试次数和上限整理成导入任务的稳定返回字段，前端无需再直接猜测 `metadata` 语义。
- 当前 TypeScript API 已补充 `POST /v1/projects/:projectId/import-tasks/upload`，首版支持上传 JSON 数组或 JSONL/NDJSON 文本文件并自动生成导入批次。
- 当前 TypeScript API 已在上传导入批次时补充解析概览、条目预览与重试历史元数据，用于支撑批次详情下钻视图。
- 当前前端已补充 `projects/:projectId/inbox` 待办页，作为工作台摘要继续处理的首版落点页面。
- 当前前端已补充 `projects/:projectId/reviews` 审核处理页，用于把审核待办从只读列表推进到可执行动作。
- 当前前端已补充 `projects/:projectId/process-documents` 过程单据处理页，用于把过程单据从只读列表推进到可执行动作。
- 当前前端已补充 `projects/:projectId/jobs` 任务状态页，用于同时展示正式导入任务和异步任务，并继续支持失败任务重试。
- 当前前端任务状态页已补充导入任务的来源文件、批次和失败明细展示，用于支撑首版导入运营视角。
- 当前前端任务状态页已补充“选择文件并创建导入批次”的入口，作为真实上传批次接入工作台的首版实现。
- 当前前端任务状态页已把导入任务改成“列表 + 批次详情”结构，方便用户查看单个上传批次的解析与失败上下文。
- 当前前端任务状态页已在批次详情中补充解析概览、条目预览和重试历史，作为导入运营视角的首版下钻面板。
- 当前前端任务状态页已补充失败原因归类、失败条目筛选、协作视角复制、协作视角清理，以及围绕失败原因的重试回流提示。
- 当前前端项目工作台已补充“最近动态”时间线，首版直接复用项目审计日志展示审核、过程单据、异步任务等最近操作。
- 当前前端已为最近动态补充可点击跳转，能够从时间线回到审核处理页、过程单据处理页和任务状态页。
- 当前前端已为最近动态补充“带焦点返回”，能够在对应处理页自动选中目标审核、过程单据或任务。
- 当前前端已为最近动态补充“复制处理链接”动作，能够直接分享审核、过程单据和任务处理入口。
- 当前前端已为工作台最近动态与待办页列表项补充“打开刚复制入口”反馈，方便用户在复制后立即核对链接目标。
- 当前前端已为项目详情页、待办页和任务状态页补充“最近复制入口”记忆，允许用户在入口页之间切换后继续打开或清除最近一次协作处理入口。
- 当前前端已为批量处理回流补充“最近协作动作”记忆，可在保留一句总述的同时按需展开本轮对象清单，减少用户重新打开回流入口才能回看的成本。
- 当前前端已补充处理完成后的回流提示，能够在返回工作台或待办页时提示摘要与最近动态已刷新。
- 当前前端项目详情页、待办页和任务状态页已支持通过 `failureReason` 保留导入失败协作视角，并在相关链接、复制动作和回流路径中统一传播。
- 当前前端项目详情页和待办页已支持继续保留 `failureResourceType`、`failureAction` 等失败子集参数，使任务状态页中已经收束出来的协作处理单元在入口页和回流链路中不会退化。
- 当前前端已在项目详情页、待办页和任务状态页中补充协作视角标签、复制链接动作和手动清除动作。
- 当前前端已对 `failureReason` 提供无效值自动清理，避免脏链接在项目工作台链路中持续传播。
- 当前前端在存在协作视角时，项目详情页与待办页中的任务状态入口会默认落到失败任务视图。

这些现状说明项目工作台专题已经从纯设计基线进入首版实现和协作运营阶段，后续扩展重点将更多集中在真实业务主干而不是继续发散导航与协作壳层。
