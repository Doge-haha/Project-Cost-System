# Spec: 报表模板专题

## 概述

本 spec 定义新点 SaaS 造价系统中的报表模板专题规则，用于统一国标模板、企业模板、字段映射、模板版本和输出格式边界。

## ADDED Requirements

### Requirement: 报表模板必须区分内置国标模板和企业模板

系统必须区分内置国标模板与企业自定义模板，不能把两类模板混作同一种可任意覆盖的资源。

#### Scenario: 管理报表模板

- **GIVEN** 用户进入报表模板管理
- **WHEN** 用户查看模板来源
- **THEN** 系统应能区分内置国标模板和企业模板

### Requirement: 内置国标模板只能复制扩展不能直接覆盖

内置国标模板只能作为基线被复制扩展，不能被直接修改覆盖，以保证系统标准输出口径稳定。

#### Scenario: 编辑内置模板

- **GIVEN** 用户尝试修改系统内置国标模板
- **WHEN** 用户提交变更
- **THEN** 系统应阻止直接覆盖
- **AND** 应通过复制生成企业模板

### Requirement: 模板字段映射必须绑定标准字段来源

报表模板中的输出字段必须绑定标准业务字段或标准计算结果，不能依赖不可追溯的人工拼接值。

#### Scenario: 配置输出字段

- **GIVEN** 模板需要配置输出字段
- **WHEN** 系统保存字段映射
- **THEN** 每个字段都应能追溯到标准来源

### Requirement: 模板必须支持多格式输出

报表模板必须允许在统一模板语义下输出 Excel、PDF 和标准交换格式等多种结果，而不是为每种格式维护完全独立口径。

#### Scenario: 选择导出格式

- **GIVEN** 用户生成某类报表
- **WHEN** 用户选择输出方式
- **THEN** 系统应支持 Excel、PDF 或标准交换格式等输出

### Requirement: 模板必须与阶段和报表类型关联

模板管理必须能够表达模板适用的业务阶段和报表类型，避免同名模板在不同阶段复用时产生歧义。

#### Scenario: 按阶段选择模板

- **GIVEN** 用户在某业务阶段生成报表
- **WHEN** 系统列出可用模板
- **THEN** 应优先展示适用于该阶段和该报表类型的模板

### Requirement: 模板变更必须保留版本

模板修改后必须保留历史版本，已生成报表继续绑定原模板版本，不得因模板更新而失去历史可复现性。

#### Scenario: 更新模板

- **GIVEN** 某模板已经生成过正式报表
- **WHEN** 用户修改模板定义
- **THEN** 系统应生成新模板版本
- **AND** 历史报表仍绑定旧版本

## Implementation Notes

以下内容是当前仓库实现现状的观察，不属于规范要求本身：

- 当前仓库已具备报表导出任务骨架，并在 `apps/worker/src/jobs/report-export-worker.ts` 中支持按 `summary` 与 `variance` 两类结果导出。
- 当前实现更接近“基于汇总结果导出 JSON/下载”的最小闭环，还没有完整落地企业模板、国标模板版本和可视化模板编辑器。
- 现有文档已对国标报表、模板字段映射和输出格式给出较完整业务口径，适合作为后续专题实现基线。

这些现状说明模板规则已经清晰，但真正的模板平台能力仍主要停留在规范和后续设计阶段。
